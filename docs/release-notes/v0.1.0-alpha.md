# ApeironCode v0.1.0-alpha — CLI-only public alpha

ApeironCode is an open-source, local-first AI coding agent for terminal-based
development. This is an early alpha, actively dogfooded, and a **CLI-only**
public release: the VS Code extension and marketing website are not included.

## Highlights

- Terminal-first AI coding agent (Commander.js + Ink + React TUI).
- Repo-aware file inspection with deterministic workspace snapshots.
- Approval-first file edits (diff preview before every write).
- Approval-first shell command execution.
- File-plan based coding workflows (model proposes JSON, runtime validates →
  previews → approves → executes via the ToolRegistry).
- App build/modify workflows for static HTML/CSS/JS and framework apps.
- Pasted runtime/build error debugging.
- Feature acceptance + offline static/browser smoke heuristics (no "fake
  success").
- Provider-flexible foundation (OpenAI, Anthropic, Gemini, Groq, DeepSeek,
  OpenRouter, OpenAI-compatible, local Ollama, plus a deterministic mock).
- Clean normal-mode terminal output with debug details behind
  `APEIRONCODE_DEBUG=1` / `--verbose`.

## Known limitations

- Early alpha; interfaces and prompts are still changing.
- **CLI only** — the VS Code extension is not part of this public alpha.
- The standalone marketing website is developed in a separate project.
- Large full-stack/multi-service apps are produced as phased plans, not one
  shot, and remain experimental.
- Browser/visual validation is **heuristic/static** (DOM + CSS over the entry
  HTML and its linked assets), not full headless-browser rendering. Package/
  framework apps document this as a limitation rather than faking a pass.
- Provider/model switching UX is still improving.
- The real TTY interaction harness is still limited.
- No full autonomy is claimed — writes and commands are approval-gated.

## Install

```bash
git clone https://github.com/poolanithinreddy/ApeironCode.git
cd apeironcode
npm install
npm run build
npm link
apeironcode setup --provider mock   # try with no API key
apeironcode
```

## Safety

Reads are low-risk; file writes and shell commands require approval; dangerous
commands are blocked or escalated; secrets are redacted in output. See
[SECURITY.md](../../SECURITY.md).
